# PII and LLM-Bound Content Policy

## 기본 결정

Secure-gateway PII detector의 직접 입력은 UTF-8 text로 제한한다. 사용자 message는 Gateway가 직접
정규화하고, 지원 파일은 격리 parser가 만든 Canonical Document IR의 text projection을 입력으로
사용한다. Detector가 지원하지 않거나 안전성을 판정할 수 없는 데이터는 Agent/LLM으로 전달하지 않는다.

## 초기 지원과 비지원

지원 PII kind와 우선순위는 [`pii-detection-catalog.md`](pii-detection-catalog.md)를 기준으로 한다.
이 프로젝트 정책에서 이름은 PII 탐지 대상이 아니며 이름 span만으로 masking하지 않는다.

지원 후보:

- 정규화 가능한 UTF-8 사용자 텍스트
- 검증된 파일 profile에서 완전하게 추출된 Canonical Document IR text
- 명시적으로 정의된 PII kind
- Gateway가 생성·관리하는 masked conversation history
- FHE-Privacy가 생성한 handle 기반 tool result

초기 비지원:

- RAG와 document retrieval
- 허용 목록 밖의 파일과 검증되지 않은 format profile
- 이미지/OCR와 음성
- clipboard 자동 수집
- 외부 memory import
- Gateway가 직접 수신하지 않은 임의 workspace 파일의 prompt 삽입
- 다른 MCP server의 민감한 plaintext result
- Agent 간 raw plaintext 메시지

## 검출 pipeline

1. 입력 크기와 encoding을 검증한다.
2. Unicode와 공백/구두점을 정책에 따라 정규화한다.
3. 지원 PII detector를 실행한다.
4. span overlap, confidence와 ambiguity를 평가한다.
5. 민감값을 안전한 저장 방식으로 보관하고 marker를 발급한다.
6. structured segment를 masked text로 직렬화한다.
7. 원문 canary/known secret 잔존과 marker 무결성을 검사한다.
8. 검증된 masked envelope만 Agent에 전달한다.

파일 형식, parser coverage와 원본 저장 정책은
[`1-9. secure-file-ingress.md`](1-9.%20secure-file-ingress.md)를 따른다. PII detector는 원본 binary나
이미지를 직접 해석하지 않으며 OCR을 수행하지 않는다.

## Fail-closed 규칙

- detector 예외 또는 timeout 시 전송하지 않는다.
- 고위험 ambiguity는 사용자 확인 또는 거부로 처리한다.
- 지원하지 않는 입력 kind는 조용히 text 변환하지 않는다.
- masking 후 알려진 secret이 남으면 전송하지 않는다.
- 사용자 입력에 포함된 가짜 marker를 capability로 인정하지 않는다.
- Gateway 오류 시 raw input fallback을 제공하지 않는다.

## 보안 주장 제한

`LLM plaintext zero`는 secure-gateway가 통제하는 지원 입력과 등록된 secret에 대한 전송 주장을
의미한다. 다음 정보까지 숨긴다고 주장하지 않는다.

- marker type과 개수
- 같은 handle의 반복 관계
- 주변 비민감 문맥
- 메시지 길이와 timing
- 공개된 operation과 access pattern
- detector가 지원 대상으로 정의하지 않은 정보

완전한 익명성 또는 모든 개인정보의 자동 검출을 주장하지 않는다.

## LLM-bound 방어 심화

- OpenShell에서 직접 provider endpoint를 허용하지 않는다.
- 승인된 inference route만 사용한다.
- 가능한 경우 inference 직전 known-secret/canary 잔존 검사를 추가한다.
- 이 검사는 인코딩·변형된 데이터의 완전한 DLP가 아니라 보조 방어로 표시한다.

## 추후 검토

- 카탈로그 kind별 locale/issuer 지원 범위
- detector 조합과 confidence 기준
- 사용자 정의 secret 등록 방식
- ambiguity 확인 UX
- conversation history 수명과 재마스킹
- attachment/이미지/음성 지원 여부
