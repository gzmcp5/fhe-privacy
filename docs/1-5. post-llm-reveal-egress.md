# Post-LLM Reveal and Terminal Egress

## 기본 결정

Agent 응답은 unresolved 상태로 Secure Gateway에 반환한다. Plaintext는 local reveal policy가 승인한
뒤 terminal user sink에서만 생성하며 Agent, LLM, tool result 또는 conversation history로 되돌리지
않는다.

## 정상 흐름

```text
OpenShell Hermes Agent response with result handle
  -> Gateway Egress
  -> handle/provenance validation
  -> Reveal Policy
  -> Reveal Authority
  -> approved terminal sink
  -> User
```

## Reveal Policy 입력

- session과 request identity
- handle kind와 computed provenance
- plaintext 목적지
- 데이터 민감도와 reveal class
- 같은 응답의 reveal 개수
- 사용자 승인 상태
- 과도하거나 우회 성격인 요청 여부

## 기본 허용/거부

| 요청 | 기본 동작 |
|---|---|
| 낮은 위험 computed result를 로컬 사용자에게 표시 | 정책 승인 후 허용 |
| input secret handle 직접 reveal | 거부 |
| Agent/LLM/tool로 plaintext 반환 | 거부 |
| unknown 또는 다른 세션 handle | 거부 |
| provenance 없는 ciphertext | 거부 |
| 고위험 secret/credential | step-up 또는 거부 |
| 한 응답의 과도한 reveal 요청 | 거부 또는 사용자 확인 |

## Terminal sink 규칙

- reveal 결과는 Gateway가 소유한 최종 출력 경로로만 보낸다.
- plaintext를 Agent response object에 다시 삽입하지 않는다.
- reveal 이후 Agent/LLM 후속 turn으로 자동 전달하지 않는다.
- logs, metrics, exception, clipboard와 telemetry에 plaintext를 기록하지 않는다.
- streaming 응답은 marker가 완성되고 검증될 때까지 외부 출력하지 않는다.
- 출력 destination이 불명확하면 reveal하지 않는다.

## 위험도 기본값

- 낮은 위험: Local Convenience Reveal 후보
- 중간 위험: 별도 사용자 확인 또는 Smartphone Step-up 후보
- 고위험: 기본 거부 또는 Threshold FHE Reveal 후보
- PC에 남기면 안 되는 값: phone fusion/display-only 후보

상세 장치 정책은 `reveal-device-policy.md`와 `threshold-fhe-reveal.md`를 참고하되, 실제 구현 전
현재 아키텍처와 다시 정합성을 검토한다.

## Fail-closed 규칙

- Reveal Authority 연결 실패 시 unresolved 상태로 사용자에게 안전한 오류를 표시한다.
- 복호 오류 시 근사값이나 partial plaintext를 출력하지 않는다.
- marker parser 오류 시 원문 문자열 치환을 시도하지 않는다.
- 정책 또는 승인 상태를 확인할 수 없으면 거부한다.
- reveal 이후 결과를 Agent가 관찰할 수 있는 channel로 보내지 않는다.

## 추후 검토

- 자동 reveal 가능한 risk class
- 사용자 승인 UX와 batch approval
- PC terminal, GUI, smartphone별 sink abstraction
- streaming response buffering 정책
- Threshold FHE Reveal 도입 시점
- reveal audit metadata와 보존 기간
