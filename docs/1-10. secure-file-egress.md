# Secure File Egress and Local Rendering

이 문서는 Agent/LLM의 마스킹된 결과를 사용자가 요청한 파일로 생성하는 terminal local-egress 경계를
정의한다. 현재 문서는 설계 결정이며 구현 또는 검증 완료를 의미하지 않는다.

## 결정

> Agent/LLM은 평문 결과 파일을 직접 생성하거나 저장하지 않는다. Secure Gateway가 마스킹된 응답을
> 검증된 Output Document IR로 정규화하고, PC·스마트폰 2-of-2 승인을 거친 Isolated Document Renderer가
> 승인된 local file Fusion Sink로서 최종 파일을 생성한다.

## 정상 흐름

```text
Agent/LLM의 마스킹된 결과와 출력 의도
  -> Secure Gateway Egress
  -> Output Document IR 정규화·schema/policy/marker 검증
  -> Reveal Coordinator
  -> PC + phone partial decrypt/unwrap 및 사용자 승인
  -> Isolated Document Renderer / approved local file Fusion Sink
  -> destination-bound marker 복원 + 허용 format 렌더링
  -> 사용자에게 로컬 파일 전달
```

Agent가 제출한 JSON, Markdown 또는 tool argument 자체를 신뢰된 IR로 취급하지 않는다. Gateway가 허용된
schema와 출력 policy에 따라 새 Output Document IR을 만들고, 알 수 없는 node, 외부 resource, executable
content와 위조 marker를 거부한다.

## Output Document IR

Output Document IR은 입력 Canonical Document IR과 같은 공통 block/segment 개념을 사용하지만 목적과
provenance를 별도로 구분한다.

- `documentId`, session, source response와 policy version에 binding한다.
- text, heading, list, table/cell, code block 등 허용 node만 포함한다.
- 복원이 필요한 값은 plaintext가 아니라 검증된 typed handle/marker로 유지한다.
- 각 handle은 output destination, format, 위치와 reveal request에 binding한다.
- 원본 파일의 macro, embedded object, image와 metadata를 자동 복사하지 않는다.
- 입력 문서와 동일한 형식으로 출력하더라도 원본 binary를 수정하지 않고 새 파일을 렌더링한다.

## Renderer와 Fusion Sink 경계

Isolated Document Renderer는 일반 Agent tool이 아니라 승인된 local file Fusion Sink다. Renderer만 두
partial 결과를 결합해 destination-bound plaintext segment를 만들고 즉시 최종 파일에 배치할 수 있다.

- Agent sandbox, MCP Bridge와 Public Compute Worker에서 호출할 수 없다.
- Reveal Coordinator의 승인된 one-shot render request만 받는다.
- 네트워크, 일반 workspace 읽기, 외부 template/font/resource fetch와 subprocess 실행을 금지한다.
- 허용된 새 output path에만 atomic write하며 symlink/reparse point와 path traversal을 거부한다.
- 평문 IR 전체나 복호 segment를 일반 임시 파일로 저장하지 않는다.
- Renderer log, exception, metrics와 crash dump에 plaintext를 남기지 않는다.
- 렌더 완료 또는 실패 후 plaintext buffer와 partial 결과를 정리한다.

## 초기 출력 형식

| Phase | 출력 형식 | 기본 정책 |
|---|---|---|
| File Output Phase 1 | `.txt`, `.md`, `.csv`, `.tsv`, `.json` | UTF-8, deterministic serialization, format별 injection escaping |
| File Output Phase 2 | `.docx`, born-digital `.pdf` | 새 문서 생성만 허용; macro, embedded/external content와 image 제외 |

초기에는 원본 파일의 pixel-perfect layout 보존을 보안 목표로 두지 않는다. 지원하지 않는 layout이나
node가 있으면 조용히 누락하지 않고 사용자에게 거부 또는 명시적 축소 결과를 제시한다. PDF 출력은
OCR과 무관하며 Renderer가 text 기반 새 PDF를 생성한다.

## 출력 위치와 전달

- 최종 destination은 reveal 승인 전에 확정하고 request에 binding한다.
- 기본 출력은 Gateway가 소유한 local export directory 또는 사용자가 명시적으로 선택한 새 파일이다.
- 기존 파일 overwrite는 별도 명시적 승인 없이는 거부한다.
- Agent가 filename이나 절대 path를 최종 결정하지 못하게 한다.
- Filename 자체의 PII와 path 안전성을 검사하고 필요하면 Gateway가 안전한 이름을 제안한다.
- 최종 파일은 plaintext 결과물이므로 encrypted original store와 구분하고 사용자에게 보호 범위 종료를
  명확히 알린다.
- 생성된 파일을 Agent/LLM conversation, tool result, provider attachment 또는 sandbox로 되돌리지 않는다.

## Fail-closed 규칙

| 실패 또는 불확실성 | 처리 |
|---|---|
| Output IR schema/policy 오류 | 렌더 중단 |
| unknown, input-secret, cross-session 또는 위조 handle | reveal/렌더 거부 |
| destination/format/nonce/policy binding 불일치 | 거부 |
| PC 또는 phone partial/승인 누락 | 거부; single-device fallback 없음 |
| 비지원 node, macro, embedded/external content | 거부 |
| unsafe path, overwrite, symlink/reparse point | 거부 |
| renderer crash, timeout 또는 partial write | 임시 output 정리 후 실패 |
| 최종 serialization 검증 실패 | publish하지 않고 정리 |

## 완료 검증

- Agent/LLM, transcript와 tool result에는 끝까지 marker/handle만 존재한다.
- Output IR의 unknown node, forged marker와 input secret direct reveal이 거부된다.
- PC 또는 phone 한쪽만으로 결과 파일을 생성할 수 없다.
- Renderer가 network, workspace input, arbitrary path와 external resource에 접근하지 못한다.
- 허용 format별 출력이 재파싱되고 기대한 구조·marker 복원 위치와 일치한다.
- Macro, embedded object, formula injection, path traversal, overwrite와 symlink 공격이 거부된다.
- 실패 시 partial file과 plaintext 임시 파일이 남지 않는다.
- 생성 파일이 Agent/LLM/provider로 되돌아가는 경로가 없다.

## 보안 주장 제한

최종 파일은 사용자에게 승인되어 생성된 plaintext local output이며 생성 이후에는 OS 파일 권한, backup,
동기화와 사용자의 공유 행위에 영향을 받는다. FHE-Privacy는 Renderer 이전의 reveal 경계와 atomic local
delivery를 보호하지만 사용자가 내보낸 파일의 이후 유통을 암호학적으로 통제한다고 주장하지 않는다.
