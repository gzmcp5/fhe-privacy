# Hermes · OpenShell · FHE-Privacy 커맨드 비교

> 목적: 같은 Hermes Agent를 단독, OpenShell sandbox, FHE-Privacy secure gateway에서 사용할 때
> 사용자 커맨드가 어떻게 달라지는지 비교한다.
>
> 상태: Hermes와 OpenShell 커맨드는 현재 CLI를 기준으로 한 예시다. `fhe-privacy` 커맨드는 아직
> 구현되지 않은 목표 사용자 인터페이스이며 구현 단계에서 변경될 수 있다.

## 한눈에 비교

| 작업 | Hermes 단독 | OpenShell 안의 Hermes | FHE-Privacy |
|---|---|---|---|
| 초기 설정 | `hermes setup` | `openshell doctor` | `fhe-privacy init` |
| 모델 설정 | `hermes model` | `openshell inference set --provider openai --model gpt-5` | `fhe-privacy inference configure --provider openai --model gpt-5` |
| 대화형 실행 | `hermes` | `openshell sandbox create --from <hermes-image> -- hermes` | `fhe-privacy gateway run --agent hermes` |
| 일회성 질문 | `hermes chat -q "질문"` | `openshell sandbox create --no-keep --from <hermes-image> -- hermes chat -q "질문"` | `fhe-privacy chat --agent hermes -q "질문"` |
| 세션 계속하기 | `hermes --continue` | `openshell sandbox connect hermes-box` | `fhe-privacy session resume <session-id>` |
| 상태 확인 | `hermes status` | `openshell sandbox list` | `fhe-privacy status` |
| 로그 확인 | `hermes logs` | `openshell logs hermes-box --tail` | `fhe-privacy logs --session <session-id> --follow` |
| 진단 | `hermes doctor` | `openshell doctor` | `fhe-privacy doctor --agent hermes` |
| 종료 | `hermes gateway stop` | `openshell sandbox delete hermes-box` | `fhe-privacy gateway stop` |
| 업데이트 | `hermes update` | 고정 binary·image 별도 교체 | `fhe-privacy update` |

## 1. Hermes 단독 사용

```bash
# 초기 설정과 모델 선택
hermes setup
hermes model

# 대화형 실행
hermes

# 일회성 질문
hermes chat -q "이번 달 카드 사용액을 정리해줘"

# 특정 모델로 질문
hermes chat --model "anthropic/claude-sonnet-4" -q "이 문서를 요약해줘"

# 이전 대화 계속하기
hermes --continue
hermes --resume <session-id>

# 상태, 로그와 진단
hermes status
hermes logs
hermes doctor

# messaging gateway 실행과 종료
hermes gateway run
hermes gateway status
hermes gateway stop

# 업데이트
hermes update
```

```text
사용자 → Hermes → LLM
```

- 사용자가 Hermes를 직접 실행한다.
- 사용자 입력은 FHE-Privacy Secure Gateway를 거치지 않는다.
- FHE-Privacy의 PII masking, Vault 등록과 local-only reveal은 적용되지 않는다.

## 2. OpenShell 안에서 Hermes 사용

```bash
# OpenShell 상태와 환경 진단
openshell status
openshell doctor

# Hermes sandbox 생성과 대화형 실행
openshell sandbox create \
  --name hermes-box \
  --from <hermes-image> \
  -- hermes

# 일회성 질문 후 sandbox 자동 삭제
openshell sandbox create \
  --no-keep \
  --from <hermes-image> \
  -- hermes chat -q "이 문서를 요약해줘"

# resource와 policy를 지정해 실행
openshell sandbox create \
  --name hermes-box \
  --from <hermes-image> \
  --cpu 2 \
  --memory 4Gi \
  --policy hermes-policy.yaml \
  -- hermes

# sandbox와 로그 확인
openshell sandbox list
openshell sandbox get hermes-box
openshell logs hermes-box --tail

# sandbox 연결과 삭제
openshell sandbox connect hermes-box
openshell sandbox delete hermes-box
```

```text
사용자 → OpenShell sandbox → Hermes → LLM
```

- 사용자가 OpenShell로 sandbox를 만들고 그 안에서 Hermes를 실행한다.
- OpenShell은 process·filesystem·network 격리를 담당한다.
- FHE-Privacy가 없다면 PII masking, Vault 등록과 reveal policy는 자동 적용되지 않는다.
- `<hermes-image>`는 Hermes가 설치된 검증·고정 OCI image reference를 의미한다.

### Windows WSL 2 예시

```powershell
.\tools\openshell\openshell.ps1 `
  -Distro Ubuntu-26.04 `
  sandbox create `
  --name hermes-box `
  --from <hermes-image> `
  -- hermes
```

## 3. FHE-Privacy로 Hermes 사용

다음 커맨드는 목표 사용자 경험을 설명하기 위한 제안 인터페이스다.

```bash
# FHE-Privacy 초기화와 전체 진단
fhe-privacy init
fhe-privacy doctor

# Hermes adapter 검사와 설치
fhe-privacy adapter doctor --agent hermes
fhe-privacy adapter install \
  --agent hermes \
  --mode secure-gateway \
  --apply

# secure gateway와 Hermes 실행
fhe-privacy gateway run --agent hermes

# 특정 policy와 모델로 실행
fhe-privacy gateway run \
  --agent hermes \
  --policy privacy-policy.yaml \
  --model gpt-5

# Gateway를 통한 일회성 질문
fhe-privacy chat \
  --agent hermes \
  -q "010-1234-5678로 연락해줘"

# session 조회와 재개
fhe-privacy session list
fhe-privacy session status <session-id>
fhe-privacy session resume <session-id>

# redacted log와 policy 확인
fhe-privacy logs --session <session-id> --follow
fhe-privacy policy validate privacy-policy.yaml

# session과 Gateway 종료
fhe-privacy session stop <session-id>
fhe-privacy gateway stop

# 제품과 고정 dependency 업데이트
fhe-privacy update
```

```text
사용자 원문
  → FHE-Privacy Secure Gateway
  → PII masking·암호화·Vault 등록
  → OpenShell sealed sandbox
  → Hermes
  → LLM
```

- 사용자는 OpenShell이나 Hermes를 직접 실행하지 않는다.
- FHE-Privacy가 Secure Gateway, Privacy Core, OpenShell sandbox와 Hermes session을 조정한다.
- Hermes와 LLM에는 masked text와 opaque handle만 전달한다.
- reveal은 PC·스마트폰 참여와 목적지 검증 후 승인된 Fusion Sink에서만 수행한다.

## 같은 요청의 차이

```bash
# Hermes만 직접 실행
hermes chat -q "010-1234-5678로 연락해줘"

# OpenShell로 격리해서 Hermes 실행
openshell sandbox create \
  --no-keep \
  --from <hermes-image> \
  -- hermes chat -q "010-1234-5678로 연락해줘"

# 개인정보를 보호하면서 격리된 Hermes 실행
fhe-privacy chat \
  --agent hermes \
  -q "010-1234-5678로 연락해줘"
```

| 방식 | Hermes가 받는 입력 | 제공되는 보호 |
|---|---|---|
| Hermes 단독 | 사용자 입력 원문 | Hermes 자체 보안 정책 |
| OpenShell + Hermes | 사용자 입력 원문 | sandbox process·filesystem·network 격리 |
| FHE-Privacy | `{{phone:h_81bc...}}로 연락해줘` 같은 masked text | pre-LLM masking, Vault, handle 연산, local-only reveal, OpenShell 격리 |

## 핵심 요약

```text
hermes       : 사용자가 Hermes를 직접 실행
openshell    : 사용자가 sandbox를 만들고 그 안에서 Hermes를 실행
fhe-privacy  : FHE-Privacy가 개인정보 보호 경계와 격리된 Hermes를 통합 실행
```
